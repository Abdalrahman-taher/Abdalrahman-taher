## Featured Project

<div align="center">

<h3>Nexora</h3>
<sub>A multi-tenant operating system for ISPs — billing, AAA, and network enforcement, kept in sync.</sub>

<br/><br/>

<img src="https://img.shields.io/badge/Source-Private-1B1B3A?style=flat-square&labelColor=0A0A0F"/>
<img src="https://img.shields.io/badge/Docs-Public-1B1B3A?style=flat-square&labelColor=0A0A0F"/>

</div>

<br/>

```mermaid
flowchart LR
    A[Billing Event] --> B[(Outbox)]
    B --> C[Event Relay]
    C --> D[Provisioning]
    D --> E[MikroTik Fleet]
    E --> F[Reconciliation]
    F -.drift.-> D
```

<details>
<summary><b>Why this is hard</b></summary>
<br/>

Billing state lives in a database; network state lives on physical devices that can be offline, slow, or lying about their own status. Keeping the two in sync — without blocking billing on flaky hardware, or letting the network drift silently — is the core problem Nexora solves.

</details>
